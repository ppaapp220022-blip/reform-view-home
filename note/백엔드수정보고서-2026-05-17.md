## 2. toss_log JSON 직렬화 버그 수정

**문제**: `response.toString()`이 Java Map 형식(`{key=value}`)으로 저장돼 MariaDB JSON 컬럼 CHECK 제약 위반 → 409

### 수정 파일

| 파일                                    | 내용                                                                                 |
|---------------------------------------|------------------------------------------------------------------------------------|
| `service/payment/PaymentService.java` | `ObjectMapper` 주입, `toJson()` 헬퍼 추가, `response.toString()` → `toJson(response)` 교체 |

```java
// rawResponse를 올바른 JSON으로 직렬화
.rawResponse(toJson(response))

private String toJson(Object obj) {
    try {
        return objectMapper.writeValueAsString(obj);
    } catch (Exception e) {
        throw new RuntimeException("toss_log JSON 직렬화 실패: " + e.getMessage(), e);
    }
}
```

## 6. 채팅방 자동 생성 + 거래 상태별 시스템 메시지

**기획**: 구매 요청 시 채팅방 자동 생성 및 안내 메시지, 거래 상태 변경 시 채팅방에 SYSTEM 메시지 자동 발송

### 수정 파일

| 파일                                    | 내용                                                                               |
|---------------------------------------|----------------------------------------------------------------------------------|
| `service/chat/ChatService.java`       | `SimpMessagingTemplate` 주입, `sendSystemMessage(postId, buyerId, content)` 메서드 추가 |
| `service/trade/TradeServiceImpl.java` | `ChatService` 주입, 4개 상태 변경 지점에서 시스템 메시지 발송                                       |
| `service/payment/PaymentService.java` | `ChatService` 주입, 결제 완료 시 시스템 메시지 발송                                             |

### sendSystemMessage 동작

1. `postId + buyerId`로 채팅방 조회 (없으면 자동 생성)
2. `type=SYSTEM`, `sender=판매자` 로 `ChatMessage` 저장
3. `/sub/chat/{chatId}` 채널로 WebSocket push

### 시스템 메시지 발송 시점

| 이벤트                      | 메시지                                   |
|--------------------------|---------------------------------------|
| 구매 요청 (`addTrade`)       | "구매 요청이 시작되었습니다. 판매자의 수락을 기다리고 있습니다." |
| 거래 수락 (`acceptTrade`)    | "판매자가 거래를 수락했습니다. 결제를 진행해 주세요."       |
| 결제 완료 (`confirmPayment`) | "결제가 완료되었습니다. 판매자가 배송 정보를 등록할 예정입니다." |
| 배송 시작 (`startShipping`)  | "판매자가 배송을 시작했습니다. 택배사/송장번호 포함"        |
| 구매 확정 (`confirmTrade`)   | "구매가 확정되었습니다. 거래가 완료되었습니다."           |
