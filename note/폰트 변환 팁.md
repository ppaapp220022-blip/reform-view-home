# 웹 폰트 변환 및 최적화 설정 가이드

웹 프로젝트(React/Vite)에서 성능과 가독성을 모두 잡기 위한 폰트 변환 팁.

---

## 1. 추천 변환 사이트

가장 안정적이고 설정이 디테일한 **Transfonter** 추천.

- URL: https://transfonter.org/

---

## 2. 최적의 설정법 (Transfonter 기준)

| 설정 항목 | 권장 값 | 이유 |
|---|---|---|
| Formats | WOFF2 | 최신 브라우저 표준, 압축률이 가장 뛰어남 |
| Family support | On | Bold, Light 등을 하나의 패밀리로 묶어 관리 가능 |
| Add local rule | On | 사용자 PC에 설치된 경우 중복 다운로드 방지 |
| Fix vertical metrics | On | 브라우저별로 폰트 높이가 달라지는 현상 방지 |
| Font display | swap | 폰트 로딩 중에도 텍스트를 보여주어 사용자 경험 개선 |
| Base64 encode | Off | 폰트 용량이 크므로 별도 파일로 관리하는 것이 유리 |

---

## 3. 파일 형식 선택 (TTF vs OTF)

웹 폰트 소스로는 **TTF**가 더 유리함.

- **TTF 추천**: 웹 브라우저 호환성이 더 높고 렌더링 안정성이 뛰어남
- **OTF**: 인쇄/출판용 고해상도 작업에 적합하나, 웹 변환 시 간혹 뭉개질 수 있음

---

## 4. 프로젝트 적용 방법

### ① 파일 배치

변환된 `.woff2` 파일을 아래 경로에 저장.

```
public/fonts/
또는
src/assets/fonts/
```

### ② index.css 선언

```css
@font-face {
  font-family: 'YourFontName';
  src: url('/fonts/yourfont.woff2') format('woff2');
  font-weight: normal;
  font-style: normal;
  font-display: swap;
}

@layer base {
  html {
    font-family: 'YourFontName', system-ui, sans-serif;
  }
}
```
